# Ledgerline Experiment Rubric

Score each dimension from 1–5. The maximum total is 30. Baseline and final use the same requirements and content.

| Dimension | Baseline | Baseline evidence | Final | Final evidence |
|---|---:|---|---:|---|
| Information hierarchy | 2 | Identical cards flatten quickstart, workflow, and reference priority. | 5 | The first-payment path leads into a distinct endpoint ledger, then state, refund, webhook, error, and reference chapters. |
| Product specificity | 2 | Realistic IDs and payloads appear inside styling that could represent any API. | 5 | Payment states, request evidence, ledger rules, IDs, amount, currency, and response transitions form the visual system. |
| Navigation and findability | 2 | Desktop anchors work; mobile places every link before the document without contextual control. | 4 | Grouped anchors, numbered chapters, endpoint jump, compact tablet rail, and mobile TOC work; there is no search or active-section tracking. |
| Responsive recomposition | 1 | Mobile stacks the sidebar and article while retaining a separately scrolling desktop parameter table. | 5 | Four tested widths switch among three-region desktop, compact rail, article/code split, and mobile drawer with record-style parameters. |
| Accessibility | 3 | Semantic headings, labeled nav, focus treatment, and readable contrast are present; interaction lifecycle is absent. | 4 | Semantic landmarks, visible focus, Escape/focus return, live copy feedback, reduced motion, and 44 px mobile controls are verified; no external assistive-technology study was run. |
| Implementation quality | 2 | Valid dependency-free HTML/CSS works but duplicates content and has no interaction tests. | 5 | Typed deterministic content, isolated clipboard behavior, static export, six unit tests, four Playwright viewport projects, and console/overflow assertions pass. |
| **Total** | **12/30** | Conventional but usable generic baseline. | **28/30** | Product-specific documentation with tested responsive behavior. |

## Method limits

This is one documented Codex/GPT-5 run, not a statistically representative evaluation. Scores are evidence-based judgments, not usability-test results. Ledgerline is fictional and none of its API behavior is production payment guidance.
