API DOCUMENTATION

Build payments with Ledgerline

A fictional REST API for creating payments, issuing refunds, and receiving webhook events. Illustrative only—not production payment guidance.

Getting Started

Use the test environment and send your API key as a bearer token.

curl https://api.ledgerline.test/v1/payments \
  -H "Authorization: Bearer ll_test_example" \
  -H "Content-Type: application/json" \
  -d '{"amount":2500,"currency":"thb","capture_method":"manual"}'

Payments

POST /v1/payments

Create a payment in test mode. The example returns pay_01J9AZ7J6Y3K2E8M4T with status requires_capture.

ParameterTypeRequiredDescription
amountintegerYesMinor currency units.
currencystringYesLowercase currency code.
capture_methodstringNoautomatic or manual.
 
{"id":"pay_01J9AZ7J6Y3K2E8M4T","amount":2500,"currency":"thb","status":"requires_capture"}

Refunds

Create a refund for a captured payment with POST /v1/refunds, then inspect it with GET /v1/refunds/:id.

Webhooks

Verify the illustrative Ledgerline-Signature header before processing payment.captured or refund.created. Failed deliveries retry in test mode.

Errors

Errors include a stable code, human-readable message, and optional param. A missing amount returns HTTP 400 with parameter_missing.

API Reference

Base URL: https://api.ledgerline.test/v1. Retrieve, capture, or cancel a payment using its payment ID.
